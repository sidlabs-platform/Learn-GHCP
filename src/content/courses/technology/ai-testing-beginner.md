# 🟢 AI-Powered Testing: Generate Tests with Copilot

Learn how to use GitHub Copilot to automatically generate unit tests, understand common test patterns, and dramatically improve your code coverage — all with less manual effort.

## Prerequisites

- GitHub Copilot active in your IDE (VS Code recommended)
- Basic familiarity with JavaScript/TypeScript or Python
- A project with at least one module or function to test
- Node.js 18+ (for Jest examples) or Python 3.10+ (for pytest examples)

## Why Testing Matters

Every line of untested code is a silent risk. Tests catch bugs before users do, document expected behavior, and give you confidence to refactor. But writing tests is often tedious — which is exactly where Copilot shines.

| Without Copilot | With Copilot |
|-----------------|--------------|
| Manually write every assertion | Generate assertions from function signatures |
| Guess at edge cases | Copilot suggests edge cases you might miss |
| Copy-paste boilerplate | Instant test scaffolding |
| 30+ minutes per module | 5–10 minutes per module |

## Understanding Test Patterns

Before generating tests, it helps to know what Copilot is producing. Here are the three patterns you'll see most often:

### Unit Tests

Test a single function in isolation. No network calls, no database — just input → output.

```javascript
// Function under test
function add(a, b) {
  return a + b;
}

// Unit test generated by Copilot
describe('add', () => {
  it('should return the sum of two positive numbers', () => {
    expect(add(2, 3)).toBe(5);
  });

  it('should handle negative numbers', () => {
    expect(add(-1, -2)).toBe(-3);
  });

  it('should return 0 when both arguments are 0', () => {
    expect(add(0, 0)).toBe(0);
  });
});
```

### Integration Tests

Test how multiple units work together — e.g., a service calling a repository.

```javascript
describe('UserService', () => {
  it('should create a user and return the saved record', async () => {
    const service = new UserService(mockRepository);
    const result = await service.createUser({ name: 'Ada', email: 'ada@example.com' });

    expect(result.id).toBeDefined();
    expect(result.name).toBe('Ada');
    expect(mockRepository.save).toHaveBeenCalledTimes(1);
  });
});
```

### Snapshot Tests

Capture the output of a component or function and compare it to a stored reference.

```javascript
it('should match the greeting snapshot', () => {
  const output = renderGreeting('Ada');
  expect(output).toMatchSnapshot();
});
```

## Step 1 — Generate Tests with the `/tests` Command

The fastest way to generate tests is with Copilot Chat's `/tests` slash command.

1. Open a source file in your editor — for example, `mathUtils.js`:

```javascript
// mathUtils.js
export function factorial(n) {
  if (n < 0) throw new Error('Negative numbers not allowed');
  if (n === 0 || n === 1) return 1;
  return n * factorial(n - 1);
}

export function isPrime(n) {
  if (n < 2) return false;
  for (let i = 2; i <= Math.sqrt(n); i++) {
    if (n % i === 0) return false;
  }
  return true;
}
```

2. Open Copilot Chat and type:

```
/tests Generate comprehensive tests for mathUtils.js
```

3. Copilot produces a full test file:

```javascript
// mathUtils.test.js
import { factorial, isPrime } from './mathUtils';

describe('factorial', () => {
  it('should return 1 for input 0', () => {
    expect(factorial(0)).toBe(1);
  });

  it('should return 1 for input 1', () => {
    expect(factorial(1)).toBe(1);
  });

  it('should return 120 for input 5', () => {
    expect(factorial(5)).toBe(120);
  });

  it('should throw an error for negative input', () => {
    expect(() => factorial(-1)).toThrow('Negative numbers not allowed');
  });
});

describe('isPrime', () => {
  it('should return false for numbers less than 2', () => {
    expect(isPrime(0)).toBe(false);
    expect(isPrime(1)).toBe(false);
  });

  it('should return true for prime numbers', () => {
    expect(isPrime(2)).toBe(true);
    expect(isPrime(7)).toBe(true);
    expect(isPrime(13)).toBe(true);
  });

  it('should return false for composite numbers', () => {
    expect(isPrime(4)).toBe(false);
    expect(isPrime(9)).toBe(false);
    expect(isPrime(15)).toBe(false);
  });
});
```

> 💡 **Tip:** Copilot reads your function signatures, JSDoc comments, and variable names to determine what tests to write. Better names = better tests.

## Step 2 — Generate Tests with Ghost Text

You don't always need Chat. Open a new test file, type the `describe` block name, and let ghost text do the rest:

```javascript
// Start typing in a new file:
import { isPrime } from './mathUtils';

describe('isPrime', () => {
  // Copilot will suggest test cases as ghost text here ✨
});
```

Press `Tab` to accept each suggestion. Copilot typically generates 3–5 test cases automatically.

## Step 3 — Python Example with pytest

Copilot works the same way in Python. Given this module:

```python
# string_utils.py
def reverse_words(sentence: str) -> str:
    """Reverse the order of words in a sentence."""
    return ' '.join(sentence.split()[::-1])

def count_vowels(text: str) -> int:
    """Count the number of vowels in a string."""
    return sum(1 for char in text.lower() if char in 'aeiou')
```

Ask Copilot Chat: `/tests Write pytest tests for string_utils.py`

```python
# test_string_utils.py
import pytest
from string_utils import reverse_words, count_vowels


class TestReverseWords:
    def test_single_word(self):
        assert reverse_words("hello") == "hello"

    def test_multiple_words(self):
        assert reverse_words("hello world") == "world hello"

    def test_empty_string(self):
        assert reverse_words("") == ""

    def test_extra_spaces(self):
        assert reverse_words("  hello   world  ") == "world hello"


class TestCountVowels:
    def test_all_vowels(self):
        assert count_vowels("aeiou") == 5

    def test_no_vowels(self):
        assert count_vowels("rhythm") == 0

    def test_mixed_case(self):
        assert count_vowels("Hello World") == 3

    def test_empty_string(self):
        assert count_vowels("") == 0
```

## Step 4 — Improve Coverage

After generating initial tests, check your coverage and ask Copilot to fill gaps:

```
What edge cases am I missing for the factorial function?
```

Copilot might suggest:

- Very large inputs (`factorial(20)`)
- Non-integer inputs (`factorial(3.5)`)
- Non-number inputs (`factorial("abc")`)

```javascript
it('should handle large inputs', () => {
  expect(factorial(20)).toBe(2432902008176640000);
});

it('should handle boundary at n=2', () => {
  expect(factorial(2)).toBe(2);
});
```

> 💡 **Tip:** Run `npx jest --coverage` or `pytest --cov` after generating tests to see exactly which lines are uncovered.

## Hands-On Exercise

**Goal:** Generate a complete test suite for an existing module.

1. Create a file called `validators.js`:

```javascript
export function isValidEmail(email) {
  const regex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  return regex.test(email);
}

export function isStrongPassword(password) {
  return (
    password.length >= 8 &&
    /[A-Z]/.test(password) &&
    /[a-z]/.test(password) &&
    /[0-9]/.test(password)
  );
}

export function isInRange(value, min, max) {
  return typeof value === 'number' && value >= min && value <= max;
}
```

2. Open Copilot Chat and type: `/tests Generate tests for validators.js with edge cases`
3. Review the generated tests — do they cover empty strings, `null`, `undefined`?
4. Run the tests: `npx jest validators.test.js --coverage`
5. If coverage is below 90%, ask Copilot: `What additional test cases would improve coverage for validators.js?`

## Troubleshooting

| Problem | Solution |
|---------|----------|
| Copilot generates tests for the wrong framework | Add a comment at the top: `// Using Jest` or `# Using pytest` |
| Tests reference non-existent imports | Verify your source file is saved and open in the editor |
| Ghost text doesn't appear in test files | Ensure Copilot extension is enabled; try restarting your editor |
| Generated tests don't match your project structure | Provide a sample test file in the same directory for Copilot to learn from |

## Glossary

| Term | Definition |
|------|-----------|
| **Unit Test** | A test that validates a single function or method in isolation |
| **Integration Test** | A test that validates how multiple components work together |
| **Code Coverage** | The percentage of your source code exercised by your test suite |
| **Ghost Text** | Copilot's inline suggestion that appears as grayed-out text in your editor |
| **Assertion** | A statement that checks whether a condition is true (e.g., `expect(x).toBe(y)`) |
| **Test Runner** | A tool that discovers and executes tests (e.g., Jest, pytest) |
